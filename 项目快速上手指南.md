## 1. 概览：定义"灵魂"

### 1.1 痛点

本项目主要解决以下传统教材平台的痛点：

1. **静态内容无法交互**
   - 传统教材中的代码示例无法直接运行，学生需要手动复制到IDE才能执行
   - 缺乏即时反馈，无法验证理解是否正确

2. **缺乏智能问答支持**
   - 学生在学习过程中遇到问题，无法获得即时、上下文相关的解答
   - 传统FAQ或论坛响应慢，且答案可能不准确

3. **内容更新困难**
   - 教材内容更新需要重新发布整个文档
   - 无法支持社区协作编辑和内容改进建议

4. **学习体验割裂**
   - 阅读、代码实践、讨论分散在不同平台
   - 缺乏统一的交互式学习环境

### 1.2 目标

我们要打造一个**交互式智能教材平台**，具备以下核心能力：

- ✅ **可执行代码块**：支持在教材中直接运行 Python/PyTorch 代码，类似 Jupyter Notebook
- ✅ **AI 智能助教**：由大模型驱动、通过系统提示词设定身份的问答助教（派派）
- ✅ **多模态交互**：支持文本+图片的视觉同步问答（Vision Sync）
- ✅ **社区协作**：支持内容块级别的讨论、编辑建议和版本管理
- ✅ **实时代码执行**：后端持久化命名空间，支持类似 Jupyter 的连续执行体验

### 1.3 现状

#### ✅ 已完成功能

**后端 (FastAPI)**
- ✅ 核心 API 端点完整实现
  - `/chat_stream`: 流式 AI 问答（系统提示词 + Qwen API）
  - `/run_code`: Python 代码执行（支持 PyTorch、matplotlib、d2l 等）
  - `/blocks`: 内容块创建
  - `/comments`: 讨论评论 CRUD
  - `/submit_suggestion`: 内容编辑建议提交
- ✅ 代码执行环境：全局命名空间持久化，支持 matplotlib 图表生成

**前端 (Next.js)**
- ✅ 教材阅读界面 (`MainContent.tsx`): 支持 Markdown 渲染、代码高亮、数学公式（KaTeX）
- ✅ 块组件 (`content/blocks/`): Jupyter 风格代码块、Markdown 块、块容器与块编辑器
- ✅ AI 聊天框 (`ChatBox.tsx`): 流式响应、视觉同步
- ✅ 讨论面板 (`ChapterDiscussion.tsx`): 章节级 / 块级评论功能
- ✅ 布局组件：左侧目录、右侧 AI/讨论面板（统一在 `TextbookPlatform.tsx` 中编排）

**数据库 (Supabase)**
- ✅ 数据模型：`books` → `sections` → `content_blocks` 三层结构
- ✅ 讨论系统：`discussions` 表支持块级评论
- ✅ 编辑建议：`edit_suggestions` 表支持内容改进建议

#### ⚠️ 待完善功能

- ⚠️ **用户认证系统**：代码中已有登录/注册路由 (`app/(auth)/login`, `app/(auth)/register`)，但具体实现需确认
- ⚠️ **内容块编辑**：UI 中有编辑按钮，但编辑逻辑可能未完全实现
- ⚠️ **建议审核流程**：`edit_suggestions` 表有 `status` 字段，但审核工作流可能未实现
- ⚠️ **多书籍支持**：目前主要针对单本书（Deep Learning - Attention Mechanisms），多书籍切换逻辑需验证
- ⚠️ **错误处理**：部分 API 调用的错误处理和用户提示可能不够完善
- ⚠️ **性能优化**：大量代码块渲染时的性能优化（虚拟滚动等）

#### 📝 技术债务

暂无，拓展功能好多写不动了

### 2.3 数据流向

#### 核心功能 1: 用户提问 AI（系统提示词 + 流式回答）

```
┌─────────────┐
│   前端 UI    │
│  ChatBox.tsx │
└──────┬──────┘
       │ 1. 用户输入问题 + 可选图片 + 历史
       │ POST /chat_stream
       ▼
┌─────────────────┐
│   FastAPI 后端   │
│   RAG.py        │
│   stream_chat   │
└──────┬──────────┘
       │ 2. 按语言构建身份系统提示词（派派的身份与职责）
       │ 3. 拼装 messages（系统提示词 + 历史 + 当前问题/图片）
       │ 4. 调用 Qwen API（流式）
       ▼
┌─────────────────┐
│   Qwen API      │
│   (外部服务)     │
└──────┬──────────┘
       │ 5. 流式返回 token
       ▼
┌─────────────────┐
│   FastAPI 后端   │
│   (流式转发)     │
└──────┬──────────┘
       │ 6. NDJSON 返回: {"type": "delta", "content": "..."}
       │ 7. 最后发送: {"type": "done", "references": []}
       ▼
┌─────────────┐
│   前端 UI    │
│  ChatBox.tsx │
│  (实时更新)   │
└─────────────┘
```

```
┌─────────────────────┐
│      前端 UI         │
│  JupyterCodeCell    │
│  (Jupyter 风格)      │

           │ 1. 用户点击"运行"按钮
           │ 2. 提取代码内容
           │ POST /run_code

┌─────────────────────┐

│   返回执行结果       │
│   {                 │

│     error: null,    │
│     plotBase64: "...",│
│     success: true   │
│   }                 │
└──────────┬──────────┘
           │
           ▼
┌─────────────────────┐
│      前端 UI         │
│  (显示输出和图表)     │
└─────────────────────┘
```

## 4. 协作规范：制定"法典"

### 4.1 代码风格

#### TypeScript/JavaScript 规范

- **类型检查**: TypeScript 严格模式 (`strict: true` 在 `tsconfig.json` 中)
- **代码检查**: ESLint（使用 Next.js 官方配置）
  - 配置文件: `web/eslint.config.mjs`
  - 运行检查: `npm run lint`
- **命名规范**:
  - 组件: PascalCase (`ChatBox.tsx`, `MainContent.tsx`)
  - 函数/变量: camelCase (`handleSend`, `isLoading`)
  - 常量: UPPER_SNAKE_CASE（如 `API_URL`）
  - 类型/接口: PascalCase (`Message`, `ChatBoxProps`)

#### Python 规范

- **代码检查**: 未发现配置 Black/Flake8，建议添加
- **命名规范**:
  - 函数/变量: snake_case (`create_block`, `user_query`)
  - 类: PascalCase (`ChatSchema`, `ContentBlock`)
  - 常量: UPPER_SNAKE_CASE (`SUPABASE_URL`, `GLOBAL_NAMESPACE`)
- **类型提示**: 部分函数使用了类型提示（`Optional[str]`, `List[ContentBlock]`），建议全面使用

### 4.2 现有规范

#### 注释风格

**前端 (TypeScript/React)**:
- 组件使用 JSDoc 风格注释（较少，建议补充）
- 关键逻辑有行内注释（如 `ChatBox.tsx` 中的视觉同步逻辑）

**后端 (Python)**:
- 函数使用 docstring（如 `routers.py` 中的 API 端点）
- 关键逻辑有行内注释（如 `RAG.py` 中的代码执行沙箱与安全审计）

## 附录

### A. 常见问题

**Q: 后端启动失败？**  
A: 检查 `backend/.env` 是否正确配置了 `SUPABASE_URL` / `SUPABASE_KEY` / `QWEN_API_URL` / `QWEN_API_KEY`，以及依赖是否已 `pip install -r requirements.txt`。

**Q: 前端无法连接到后端？**  
A: 检查后端是否在 `http://localhost:8000` 运行，以及前端环境变量 `NEXT_PUBLIC_API_URL` 是否正确配置。

**Q: AI 助教不回复 / 报错？**  
A: 检查 `QWEN_API_URL` 与 `QWEN_API_KEY` 是否有效（助教直接转发到 Qwen API，不再依赖本地检索）。

**Q: 代码执行失败？**  
A: 检查后端是否安装了所有必需的 Python 包（torch, numpy, pandas, matplotlib, d2l 等）。
